## What Chronicle Stack Can Safely Assume

- downstream consumers now have a separate implementation repo that can test
  Chronicle-produced query-engine bundles without asking Chronicle core to host
  the runtime
- the authoritative boundary remains unchanged:
  Chronicle primary records and Chronicle-side completion decisions stay in
  `chronicle-stack`
- plugin-backed answers are review support only and do not become
  Chronicle-authoritative facts
- any future write-back requirement must return to explicit Chronicle-side
  architecture review first

## Recommended Upstream Interpretation

- treat `chronicle-external-query` as the current reference downstream runtime
  workspace for query-engine handoff bundle sufficiency testing
- keep Chronicle-side scope focused on producing stable handoff contracts and
  representative bundles
- do not move plugin runtime concerns, hosted provider credentials, or
  downstream evaluation artifacts back into Chronicle core
