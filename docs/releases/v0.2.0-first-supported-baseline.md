# v0.2.0 First Supported Baseline

`v0.2.0` is the first supported local downstream runtime baseline for
`chronicle-external-query`.

## What Is Released

- Chronicle handoff bundle ingest with required-file, required-key,
  contract-version, and cross-file consistency validation
- graph-only and hybrid retrieval with shared provenance, deterministic local
  vector fixtures, and representative-bundle regression coverage
- runtime answers with explicit status, provenance, and reviewable metadata
- evaluation artifacts that can be saved, reloaded, compared, and rendered as
  markdown trial or comparison reports
- a reproducible clean-checkout smoke path used by both operators and CI

## Supported Boundary

- local-only execution
- no Chronicle write-back
- no hosted query runtime
- no hosted graph runtime
- no hosted vector provider required for the supported path
- Chronicle `.chronicle/chronicle.jsonl` remains authoritative over any
  downstream-derived output

## Completion Evidence

- Milestones A-D are closed
- representative sanitized bundle fixtures are committed
- full local test suite passes
- clean-checkout smoke passes from both the working checkout and a fresh copy
- CI is wired to the same smoke path

## Operator Entry Point

The supported operator flow is:

```bash
bash scripts/smoke_clean_checkout.sh
```

If the default `python3` is older than 3.11:

```bash
PYTHON_BIN=/usr/local/bin/python3.11 bash scripts/smoke_clean_checkout.sh
```
